# PreschoolClubsApp — система учёта кружков дошкольного развития

**Тип приложения:** консольное (CLI) на C++17.  
**Назначение:** автоматизация хранения, поиска и редактирования информации о кружках в детских садах и центрах развития.  

Приложение позволяет:
- вести единый справочник кружков (вид, название, руководитель, расписание, время, стоимость, количество учеников);
- разграничивать доступ: **администратор** может всё (управлять пользователями, добавлять/менять/удалять кружки, выполнять массовые операции), **пользователь** — только просмотр, поиск, сортировку и выполнение двух заданных отчётов;
- хранить данные в простых текстовых файлах (`users.txt` для учёток, `clubs.txt` для кружков) — не требует установки базы данных;
- работать "из коробки" на Windows (поддержка UTF-8 в консоли).

## 📋 Функциональные возможности (подробно)

### 1. Авторизация и роли
- При запуске запрашивается **логин** и **пароль**.
- Если пара не найдена — программа завершается с сообщением об ошибке.
- Если найдена — определяется роль (`admin` или `user`) и открывается соответствующее меню.
- **Дефолтный администратор:** логин `admin`, пароль `admin` (создаётся автоматически при первом запуске).

### 2. Возможности администратора

#### 2.1 Управление учётными записями пользователей
- **Просмотр** всех учёток (логин, пароль, роль) в виде таблицы.
- **Добавление** нового пользователя (логин должен быть уникальным, иначе ошибка).
- **Редактирование** существующего пользователя (можно изменить логин, пароль, роль).
- **Удаление** пользователя (с подтверждением `y/n`).

#### 2.2 Редактирование данных о кружках (CRUD)
- **Просмотр** всех кружков в форматированной таблице: номер, вид, название, ФИО руководителя, график, время, цена, количество учеников.
- **Добавление** нового кружка (все поля вводятся с клавиатуры; цена и количество учеников не могут быть отрицательными).
- **Удаление** кружка по номеру (с подтверждением).
- **Редактирование** кружка по номеру (можно изменить любое поле, оставив старое значение, если ввести пустую строку).

#### 2.3 Обработка данных (индивидуальные задания и аналитика)
- **Индивидуальное задание №1:** удалить все кружки, где количество учеников **меньше** заданного значения. Перед удалением показывает список таких кружков и запрашивает подтверждение.
- **Индивидуальное задание №2:** изменить график проведения у выбранного кружка (выбирается по номеру, вводится новое расписание).
- **Поиск** кружков по четырём критериям:
  - по виду кружка (частичное совпадение, без учёта регистра? на самом деле как есть, но можно писать часть слова);
  - по названию (частичное совпадение);
  - по ФИО руководителя (частичное совпадение);
  - по стоимости (диапазон от минимальной до максимальной цены).
- **Сортировка** списка кружков (с сохранением в файл) по:
  - названию (алфавитный порядок);
  - стоимости (по возрастанию);
  - количеству учеников (по возрастанию);
  - виду кружка (по алфавиту).

> Все изменения (добавление, удаление, редактирование, сортировка) **автоматически сохраняются** в файл `clubs.txt`.

### 3. Возможности пользователя

Пользователь **не может** изменять данные — только читать, искать и выполнять два предопределённых отчёта.

#### 3.1 Просмотр и аналитика
- **Просмотр всех кружков** (такая же таблица, как у администратора).
- **Индивидуальное задание №1:** вывести все сведения о кружках **определённого вида** (точное совпадение).
- **Индивидуальное задание №2:** вывести все сведения о кружках **выбранного руководителя** (точное совпадение).

#### 3.2 Поиск (без возможности изменения)
- **По названию** (частичное совпадение).
- **По диапазону стоимости** (от – до).

#### 3.3 Сортировка (временная, без сохранения в файл)
- По названию (алфавит);
- По стоимости (возрастание);
- По количеству учеников (возрастание).

Результаты сортировки показываются на экране, но исходный файл `clubs.txt` **не перезаписывается** (в отличие от администраторской сортировки).

## 🛠 Технические детали

**Язык и стандарт:** C++17.  
**Используемые библиотеки:**  
`iostream`, `fstream`, `vector`, `string`, `algorithm`, `iomanip`, `sstream`, `cctype`, `limits`, `windows.h` (только для настройки кодировки CP_UTF8).  

**Структуры данных:**
- `struct User` — логин, пароль, роль.
- `struct Club` — вид, название, директор, график, время, цена, количество учеников.

**Файловое хранение:**
- `users.txt`: каждый пользователь на отдельной строке через пробел: `login password role`
- `clubs.txt`: каждая запись через разделитель `|` в одной строке:  
  `вид|название|директор|график|время|цена|ученики`

При запуске данные загружаются в `std::vector`, изменения пишутся обратно в файлы.

**Валидация ввода:**  
- Цена и количество учеников проверяются на `>=0`.  
- При вводе чисел с неверным форматом программа запрашивает повторно.  
- При удалении/редактировании запрашивается подтверждение.

## 🚀 Как скомпилировать и запустить

### Требования
- Windows (для корректного отображения кириллицы в консоли используется `SetConsoleOutputCP(CP_UTF8)`).
- Компилятор с поддержкой C++17 (Visual Studio 2019/2022, MinGW с `-std=c++17`, Clang).

### Компиляция в Visual Studio
1. Открыть `.cpp` файл в Visual Studio.
2. Собрать решение (Ctrl+Shift+B).
3. Запустить без отладки (Ctrl+F5).

```cmd
cl /EHsc /std:c++17 PreschoolClubsApp.cpp
